Instalación

Requisitos del Sistema

  • PHP 7.4 o superior
  • MySQL 5.7+ o MariaDB 10.3+
  • Apache con mod_rewrite o servidor PHP integrado
  • Extensiones: PDO, PDO_MySQL, JSON, mbstring, OpenSSL, libxml, cURL

1. Configurar Base de Datos

Editar config/database.php con las credenciales:

return [
    'host' => 'localhost',
    'database' => 'facturacion_electronica',
    'username' => 'usuario',
    'password' => 'contraseña',
    'charset' => 'utf8mb4'
];

2. Crear Base de Datos

mysql -u usuario -p < database.sql

O usar el script de instalación:

php install.php

3. Iniciar Servidor

php -S localhost:8000 -t .

API Endpoints

Todos los endpoints devuelven JSON. La mayoría requieren autenticación Bearer token.

Autenticación

POST /api/register

Registro de empresa y usuario administrador

{
  "company": {
    "business_name": "Empresa S.A.S",
    "nit": "900123456-7",
    "tax_regime": "Común",
    "ciiu_code": "6201"
  },
  "user": {
    "first_name": "Juan",
    "email": "admin@empresa.com",
    "password": "contraseña",
    "document_type": "CC",
    "document_number": "1234567890"
  }
}
POST /api/login

Inicio de sesión. Devuelve token Bearer

{
  "email": "admin@empresa.com",
  "password": "contraseña"
}

Respuesta:

{
  "success": true,
  "data": {
    "token": "bearer_token_aqui",
    "user": { ... }
  }
}
GET /api/me

Obtener información del usuario autenticado

Requiere: Header Authorization: Bearer {token}

Sistema de Webhooks

Notificaciones HTTP POST automáticas para eventos del sistema.

Eventos Disponibles

  • invoice.created - Factura creada
  • invoice.accepted - Factura aceptada por DIAN
  • invoice.rejected - Factura rechazada
  • invoice.cancelled - Factura cancelada
  • certificate.expiring - Certificado próximo a vencer
  • certificate.expired - Certificado vencido
POST /api/webhooks

Crear webhook

{
  "event_type": "invoice.accepted",
  "url": "https://mi-sistema.com/webhook",
  "secret": "mi-secreto-opcional",
  "headers": {
    "Authorization": "Bearer token"
  }
}

Estructura del Payload

{
  "event": "invoice.accepted",
  "timestamp": "2024-12-13 20:00:00",
  "data": {
    "invoice_id": 123,
    "invoice_number": "FAC-1-8",
    "cufe": "...",
    "status": "accepted"
  }
}

Headers enviados:

  • Content-Type: application/json
  • X-Webhook-Event: {event}
  • X-Webhook-Signature: {hmac_sha256}

Validación XSD

El sistema valida automáticamente todos los documentos XML contra los esquemas XSD oficiales de UBL 2.1 antes de enviarlos a DIAN.

Proceso de Validación

  1. Generación del XML UBL 2.1
  2. Validación estructural básica
  3. Validación contra esquema XSD
  4. Detección de errores y advertencias
  5. Bloqueo de envío si hay errores

Integración con DIAN

El sistema soporta dos modos de operación: simulador (sandbox) y producción real.

Modo Simulador

Ideal para desarrollo y pruebas. No requiere certificados reales.

USE_REAL_DIAN = false

Modo Producción

Conexión real con API DIAN. Requiere certificado digital .p12

USE_REAL_DIAN = true

Configuración

Editar config/dian.php:

define('USE_REAL_DIAN', true);
define('DIAN_ENVIRONMENT', 'HABILITACION');
define('DIAN_CERTIFICATE_PATH', '/ruta/certificado.p12');
define('DIAN_CERTIFICATE_PASSWORD', 'contraseña');
define('DIAN_USERNAME', 'usuario_dian');
define('DIAN_PASSWORD', 'contraseña_dian');

Cálculo de Impuestos

El sistema calcula automáticamente los impuestos según múltiples factores.

Factores de Cálculo

  • Régimen Tributario: Simplificado, Común, Gran Contribuyente, Especial, No Responsable
  • Tipo de Comprador: Persona Natural, Jurídica, No Responsable
  • Sector Económico: Código CIIU determina reglas especiales
  • Tipo de Factura: Exportación no aplica IVA
  • Impuestos del Producto/Servicio: IVA, Retenciones configuradas
